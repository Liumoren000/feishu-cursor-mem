## 2. 项目背景与目标
### 2.1 背景
当前中医药市场存在药材来源不明、真伪难辨等问题。通过建立一套类似“码上追溯”的 Web 系统，实现从中药材种植、加工、检验、物流到销售的全过程数字化记录。

### 2.2 目标
- **消费者端**：提供便捷的扫码查询工具，增强品牌信任。
- **企业端**：实现生产流通过程的精细化管理及合规性记录。
- **监管端**：提供全生命周期的追溯数据支撑，快速定位质量问题。

## 3. 业务流程图
1. **种植基地**：采收申报 -> 赋码申请。
2. **生产工厂**：原材料入库 -> 生产加工 -> 成品赋码 -> 质检报告关联。
3. **流通环节**：仓储入库 -> 调拨出库 -> 零售终端。
4. **终端用户**：手机扫码 -> 追溯信息展示（真伪核验）。

## 4. 功能需求详情
### 4.1 企业认证与管理
- 企业入驻申请（三证合一上传）。
- 权限角色配置（管理员、操作员、审计员）。

### 4.2 产品与批次管理
- 产品档案建立（品名、执行标准、原产地）。
- 生产批次关联。

### 4.3 赋码中心 (核心)
- **一物一码**：支持唯一序列号生成。
- **码段管理**：支持与防伪标签服务商对接。
- **二维码生成**：提供高性能二维码下载接口。

### 4.4 仓储与物流
- 入库记录（扫码入库）。
- 出库管理（去向追踪）。
- 库存实时盘点。

### 4.5 追溯查询页面 (H5/Web)
- 基本信息：产地、生长周期、生产日期。
- 质量证明：质检单、溯源证书（支持区块链存证展示）。
- 物流轨迹：流转关键节点。

## 5. 非功能需求
- **安全性**：数据加密传输，支持数字签名防止篡改。
- **并发性能**：支持高峰期（如大促、集中监管查询）下的高并发扫码。
- **可扩展性**：支持未来对接政府监管平台或第三方电商平台。

## 6. 数据埋点需求
- 扫码地理位置分布统计。
- 扫码用户设备、时段分析。
- 扫码异常（假码警告）实时上报。

## 7. 验收标准 (AC)
- 扫码响应时间 < 1.5s。
- 追溯信息准确率 100%（与录入数据一致）。
- 支持导出 Excel 格式的流转记录报表。
