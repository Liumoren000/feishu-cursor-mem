## 2. 项目背景与目标
### 2.1 业务背景
中医药行业面临“良莠不齐、源头难溯、监管滞后”的痛点。消费者对中药材的道地性、农残、重金属超标等问题日益关注。本项目旨在通过 Web + H5 体系，构建一套从“田间地头”到“消费者指尖”的闭环追溯系统。

### 2.2 核心目标
1. **真实性保障**：利用一物一码技术，确保每一份药材都有唯一的身份标签。
2. **全流程透明**：记录种植、采收、加工、检测、入库、发货等全流程数据。
3. **品牌数字化**：通过 H5 溯源页向消费者展示产品故事与质量证明，提升品牌价值。
4. **监管合规**：满足国家医药监管部门对追溯体系的建设要求。

## 3. 业务逻辑与流程
### 3.1 核心流程图 (文本描述)
- **生产链**：[基地] 种子管理 -> 施肥/喷药记录 -> 采收赋码 -> [工厂] 粗加工 -> 精深加工 -> 批次质检 -> 成品关联赋码。
- **流通链**：[仓库] 入库扫描 -> 调拨确认 -> [门店/终端] 销售扫码。
- **消费链**：[消费者] 微信/Web 扫码 -> 防伪验证 -> 获取追溯详情。

### 3.2 角色模型
- **超级管理员**：系统运维、企业入驻审批、字典配置。
- **企业管理员**：企业信息维护、人员权限分配。
- **操作员**：具体业务操作（赋码、入库、检测录入）。
- **消费者**：终端扫码查询用户。
- **监管机构**：数据审计与看板查看。

## 4. 功能需求详情

### 4.1 企业工作台 (Admin Portal)
#### 4.1.1 企业档案管理
- **功能描述**：支持企业上传营业执照、生产许可证、产地证明等资质。
- **业务规则**：必须通过超级管理员审核后，企业方可进行赋码操作。
- **UI 建议**：卡片式展示资质状态，支持在线预览 PDF/图片。

#### 4.1.2 产品与批次管理
- **功能描述**：定义产品规格、成分、产地、执行标准。
- **逻辑**：赋码必须关联具体的产品批次，批次需关联质检报告。

### 4.2 数字化赋码系统 (Coding Center)
#### 4.2.1 赋码引擎
- **功能描述**：生成唯一、随机且不可预测的追溯码序列号。
- **逻辑**：
  - 支持生成“明码”与“暗码”。
  - 码结构：由企业代码(4位) + 产品类型(2位) + 时间戳(8位) + 随机校验码(4位) 组成。
- **导出功能**：支持导出 CSV 供标签印刷商直接调用。

#### 4.2.2 一物一码关联
- **功能描述**：在流水线或加工端，将物理标签与系统批次建立逻辑连接。

### 4.3 智慧仓储与流通 (Warehouse & Logistics)
#### 4.3.1 入/出库管理
- **功能描述**：支持 PDA 或手机端扫码进行批量入库/出库。
- **AC**：出库时必须记录去向（客户名称或终端位置），实现链路可追。

#### 4.3.2 动态轨迹记录
- **功能描述**：自动记录产品从 A 地到 B 地的时间、经办人。

### 4.4 消费者溯源门户 (Consumer Portal - H5)
#### 4.4.1 防伪验证
- **逻辑**：首次扫码显示“正品，首次查询”；后续扫码显示“查询次数：N次，首次查询时间：YYYY-MM-DD”，提示防伪风险。
#### 4.4.2 追溯全景展示
- **内容模块**：
  - **身份卡片**：产品名称、图片、溯源 ID。
  - **成长日记**：展示产地监控图、施肥时间轴、采收视频（如有）。
  - **硬核质检**：展示三方检测机构的检测报告。
  - **流转信息**：展示简化的物流时间轴。
- **UI 建议**：清新绿色调，符合中医药文化氛围。

### 4.5 监管看板 (Dashboard)
- **核心指标**：企业分布图、扫码活跃度、假码预警分布、产品热力图。

## 5. 技术需求与约束
### 5.1 数据架构建议
- **主数据库**：MySQL (存储结构化业务数据)。
- **缓存**：Redis (存储高频扫码防伪逻辑)。
- **文件存储**：OSS (存储质检报告、企业资质图片)。
- **存证支持 (可选)**：接入区块链 API (如蚂蚁链)，将关键节点哈希存证。

### 5.2 接口需求
- **二维码解析接口**：QPS 需支持 1000+。
- **赋码下载接口**：支持单次百万级数据的生成与导出。

---

## 6. 非功能性需求 (Quality Attributes)
- **性能**：扫码页面首屏加载时间 < 1.0s。
- **安全**：
  - 接口防爬：防止竞品爬取所有码段数据。
  - 数据脱敏：敏感物流信息对普通消费者隐藏部分字段。
- **兼容性**：支持主流微信内置浏览器、Safari、Chrome。

---

## 7. 验收标准 (Acceptance Criteria)
1. **完整性**：从赋码到扫码的链路完整闭环。
2. **准确性**：扫码展示的数据与后台录入数据完全一致。
3. **鲁棒性**：并发导出 50 万码段时，系统不出现 OOM。
4. **防伪性**：同一码多次扫码能准确提示查询频次。
