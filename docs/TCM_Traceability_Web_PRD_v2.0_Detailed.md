## 2. 项目概述
### 2.1 背景与目标
针对中医药行业药材来源不透明、流通环节监管难等现状，本项目旨在构建一个连接种植、加工、物流与消费者的全链路追溯平台。通过“一物一码”技术，实现产品来源可查、去向可追、责任可究。

### 2.2 目标用户
- **B 端企业**: 种植基地、生产加工厂、物流公司、经销商。
- **C 端消费者**: 终端购药者或养生爱好者。
- **监管机构**: 中医药管理局、市场监管部门。

## 3. 业务流程与逻辑
### 3.1 核心业务流
1. **资质审核**: 企业上传营业执照、GMP/GSP 证书。
2. **产品建档**: 定义产品属性（如：道地药材产地、规格）。
3. **批次生产**: 关联种植/采收记录，录入加工参数。
4. **赋码分发**: 算法生成唯一加密码，支持单枚打印或卷标打印。
5. **节点采集**: 物流入库/出库扫码，自动更新流转状态。
6. **消费者查验**: H5 页面展示完整溯源档案及防伪结果。

### 3.2 防伪逻辑详解 (Anti-Counterfeit Logic)
- **首次扫码**: 系统识别为“首次查询”，显示绿色“正品保证”标识，记录扫码地理位置和时间。
- **多次扫码**:
    - 第 2 次及以上查询：显示橙色“多次扫码”预警。
    - 界面提示：“该码已被查询过 N 次，首次查询时间：YYYY-MM-DD HH:MM:SS，扫码地点：XXX。若非本人操作，请谨防造假。”
- **异常限制**: 若同一码在短时间内跨地域频繁查询（如 10 分钟内两地跨省），系统自动标记为“异常疑似造假”，并向企业端推送预警。

## 4. 功能需求详情

### 4.1 B 端管理门户 (Enterprise Portal)

#### 4.1.1 企业与资质管理
- **字段**: 企业名称、统一社会信用代码、法人信息、生产许可证书（图片）、GSP 证书（图片）。
- **功能**: 资质过期提醒，分级权限控制（管理员、操作员、质检员）。

#### 4.1.2 溯源档案录入 (核心模块)
- **种植模块数据项**:
    - `seed_source`: 种子/种苗来源。
    - `planting_date`: 种植日期。
    - `soil_type`: 土壤类型（如：红壤、沙质土）。
    - `fertilizer_record`: 施肥记录（品牌、成分、日期）。
    - `pesticide_record`: 用药记录（药名、浓度、休药期）。
    - `harvest_date`: 采收日期。
- **加工模块数据项**:
    - `drying_method`: 干燥方式（阴干、烘干、晒干）。
    - `temp_control`: 温度控制参数（℃）。
    - `slicing_spec`: 切片规格（厚度 mm）。
    - `steam_time`: 蒸制时长（min）。
- **质检模块数据项**:
    - `heavy_metals`: 重金属含量（Pb, Cd, Hg 等）。
    - `residue_analysis`: 农药残留检测结果。
    - `active_content`: 有效成分含量（如：黄芪甲苷含量）。

#### 4.1.3 追溯码生成与管理
- **生成规则**: `Prefix (2 chars)` + `Timestamp (6 chars)` + `Random Salt (6 chars)` + `Check Digit (2 chars)`，Base62 编码。
- **操作**: 支持批量导出（CSV/Excel/JSON）、关联打印设备。

### 4.2 C 端 H5 溯源页 (Consumer Portal)

#### 4.2.1 页面构成
- **头部 (Header)**: 产品名称、规格、批次号。
- **防伪状态栏**: 实时显示查询次数、首次查询信息。
- **全生命周期时间轴 (Timeline)**:
    - 种植阶段（产地地图展示、气象数据）。
    - 采收加工（车间照片、加工员签名）。
    - 检验报告（PDF/图片弹框）。
    - 物流信息（流转轨迹）。
- **品牌展示区**: 企业 VR 宣传视频（可选）、荣誉证书。

## 5. UI/UX 设计建议

### 5.1 B 端 Dashboard (桌面端)
- **布局**: 左侧导航栏，右侧卡片式统计区。
- **统计组件**: 实时赋码曲线、库存批次健康度环形图、扫码热力图（基于消费者扫码定位）。
- **交互**: 录入档案采用“步骤条 (Steps)”引导，降低复杂表单的认知负担。

### 5.2 C 端 H5 (移动端)
- **视觉风格**: 采用“中医古风”与“科技感”结合的配色（墨绿/土黄/米白）。
- **动效**: 扫码成功后，防伪结果卡片由下往上平滑弹出。
- **交互**: 时间轴节点点击可展开详细数据，图片支持点击放大预览。

## 6. 非功能性需求

### 6.1 性能要求
- **高并发支持**: 溯源码生成接口需支持 > 10,000 TPS。
- **响应时间**: 溯源页面首屏加载时间 < 1.5s，API 接口平均响应 < 200ms。
- **并发扫码**: 支持在促销活动期间百万级并发扫码请求。

### 6.2 安全性
- **数据存储**: 所有敏感字段（如质检员身份）进行脱敏处理。
- **防爬虫**: H5 溯源接口增加图形验证码/行为验证，防止恶意拖库。
- **审计日志**: 记录 B 端用户所有修改追溯档案的操作日志。

### 6.3 扩展性
- **接口标准**: 提供 RESTful API 供省级、国家级中医药追溯平台对接。
- **多语言**: 系统架构支持未来扩展英/日/韩等多语言版。

## 7. 验收标准 (AC)
1. **AC-01 (防伪验证)**: 用户扫同一码 3 次，系统准确显示“查询次数：3”，且首次查询时间与实际相符。
2. **AC-02 (档案完整性)**: 在 B 端完成种植至销售的全流程录入后，C 端需能展示至少 5 个关键时间轴节点。
3. **AC-03 (码唯一性)**: 生成 1000 万个追溯码，必须保证 100% 不重复。
4. **AC-04 (兼容性)**: 在 iOS 16+ 及 Android 12+ 环境下的微信内置浏览器中，页面显示无布局错乱。
