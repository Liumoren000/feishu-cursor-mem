## 2. 项目背景与目标
### 2.1 背景
中医药行业面临真伪难辨、药材来源不透明、流通环节多等痛点。国家政策正推动“一物一码”全过程追溯体系建设。
### 2.2 目标
- **B 端 (Business)**: 提供企业入驻、产品备案、生产赋码、入出库管理的一站式平台。
- **C 端 (Consumer)**: 提供极简的扫码体验，展示产品从种植、加工、检验到流通的全链路信息。
- **监管端**: 实现数据可视化，异常预警。

## 3. 业务流程
1. **企业入驻**: 提交资质 -> 审核 -> 激活。
2. **产品备案**: 录入产品信息 -> 绑定质检标准。
3. **生产赋码**: 创建生产批次 -> 生成溯源码（唯一序列号）-> 打印/贴码。
4. **仓储物流**: 入库扫码 -> 出库扫码 -> 记录流转节点。
5. **消费者扫码**: 微信/浏览器扫码 -> 校验真伪 -> 查看溯源档案。

## 4. 功能需求详情

### 4.1 B 端：企业管理门户
- **企业工作台**: 展示赋码统计、库存预警。
- **追溯码管理**: 支持按批次生成一物一码（防伪码 + 序列码）。
- **批次溯源档案**: 维护种植记录、化肥农药使用情况、炮制加工参数。
- **质检报告 (QA)**: 上传附件或关联第三方检测数据。

### 4.2 C 端：消费者 H5 溯源页
- **真伪验证**: 首次扫码显示“正品”，多次扫码提示“重复扫码次数及时间”，警惕造假。
- **全景溯源图**: 时间轴展示（种植 -> 采收 -> 加工 -> 检验 -> 销售）。
- **企业名片**: 展示企业资质、品牌介绍。

### 4.3 管理后台：监管与运营
- **码库监控**: 追踪每一个码的生命周期。
- **预警中心**: 发现异常扫码聚集地、过期批次。

## 5. 技术约束与建议
- **赋码规则**: 采用 ID 混淆算法或 Hash 算法，确保溯源码不可预测。
- **安全性**: 采用 HTTPS 加密，接口层增加防刷限制。
- **存储**: 图片/文档存储于 OSS。

## 6. 验收标准 (AC)
- 溯源码生成速度需支持单次 10 万级。
- 消费者扫码响应时间在 1s 以内。
- 溯源页面适配主流移动端浏览器。
