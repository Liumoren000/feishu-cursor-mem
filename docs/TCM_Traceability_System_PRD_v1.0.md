## 2. 项目背景与目标
### 2.1 背景
中医药行业面临原产地造假、炮制工艺不透明、质量检测报告缺失等痛点，导致消费者信任度低。
### 2.2 目标
构建一个覆盖“种植-加工-仓储-物流-销售”全生命周期的追溯系统，解决防伪验真、质量穿透和监管透明化问题。

## 3. 业务流程图
1. **企业入驻** -> 资质审核 -> 权限开启
2. **产品建模** -> 批次创建 -> 赋码申请
3. **生产赋码** -> 产线打码 -> 数据激活
4. **仓储流转** -> 入库扫码 -> 出库扫码 -> 物流绑定
5. **消费者扫码** -> 防伪校验 -> 查看全链路溯源信息

## 4. 功能需求详情

### 4.1 B 端：企业管理工作台
#### 4.1.1 企业与资质管理
- **功能**：企业基础信息维护、GMP/GSP 证书上传。
- **逻辑**：资质到期自动预警，未上传核心资质无法进行赋码。

#### 4.1.2 产品与批次管理
- **功能**：产品档案、生产批次创建。
- **字段**：品种名称、产地（GPS坐标）、收割日期、炮制方法、质检报告（PDF/图片）。

#### 4.1.3 赋码引擎 (一物一码)
- **功能**：生成唯一的加密溯源码。
- **逻辑**：采用“企业码 + 批次码 + 流水号 + 随机防伪校验码”结构。
- **防伪**：支持查询次数监控，首次扫码标记，重复扫码警告。

#### 4.1.4 仓储流转模块
- **功能**：扫码入库/出库。
- **逻辑**：记录每一次所有权转移的时间、地点、操作人。

### 4.2 C 端：H5 溯源门户
#### 4.2.1 扫码验证
- **功能**：快速展示产品真伪状态。
- **UI**：显眼的“正品”或“可疑”标识。

#### 4.2.2 溯源时间轴
- **展示**：以时间轴形式展示：种子来源 -> 种植过程 -> 采收 -> 加工炮制 -> 质量检验 -> 仓储流转。
- **交互**：点击环节可查看图片、视频或质检证书。

#### 4.2.3 消费者互动
- **功能**：一键联系厂家、评价反馈、品牌商城跳转。

### 4.3 监管看板
#### 4.3.1 扫码热力图
- **展示**：全国范围内的扫码分布，分析消费市场动态。

#### 4.3.2 预警系统
- **逻辑**：同一码异地同时扫码、频繁扫码自动触发“疑似仿冒”告警。

## 5. 非功能需求
- **安全性**：溯源码需具备高强度随机性，防止预测。
- **性能**：C 端扫码响应时间 < 1.5s，支持高并发。
- **存储**：质检文件使用云存储（OSS），关键溯源哈希存证（区块链支持可选）。

## 6. 验收标准 (AC)
1. 消费者扫码后能清晰看到从种植到销售的完整节点。
2. 重复扫描防伪码时，系统必须弹出“该码已被扫描过”的风险提示。
3. 企业端生成的每一枚标签均能在后台追溯其对应的生产批次。
