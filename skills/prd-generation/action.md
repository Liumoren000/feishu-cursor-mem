# 需求文档（PRD）编写技能

## 概述
当用户请求编写需求文档时，应遵循以下流程以确保文档的高质量和专业性。

## 执行流程
1. **收集信息**：
   - 背景与目标：为什么要做这个？
   - 核心功能：关键卖点或解决的核心痛点。
   - 用户画像：目标用户是谁？
   - 业务流程：关键的操作链条。
2. **构建框架**：
   - 提供标准 PRD 目录。
   - 基于收集的信息填充初步内容。
3. **细化需求**：
   - 针对每个功能点进行详细描述（功能、逻辑、UI/UX 建议）。
   - 定义验收标准（AC）。
4. **生成文档**：
   - 提供 Markdown 格式的完整文档。
   - 如果内容过长，建议在仓库创建临时分支并提供链接。

## 标准模板参考
- 1. 修订记录
- 2. 项目背景与目标
- 3. 业务流程图
- 4. 功能需求详情
- 5. 非功能需求（性能、安全等）
- 6. 数据埋点需求
- 7. 验收标准
